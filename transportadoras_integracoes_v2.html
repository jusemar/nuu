Como funciona a conexão Frenet

A Frenet é o motor de cotação de fretes do sistema. Ela consulta várias transportadoras ao mesmo tempo e retorna os serviços disponíveis para cada envio.

Token configurado
CEP de origem
API Frenet ativa
Catálogo salvo

Exemplo prático

Você informa CEP destino 01310-100, peso 2 kg e dimensões 30×20×15 cm. A Frenet retorna: PAC (7 dias / R$ 18), SEDEX (2 dias / R$ 32), Jadlog Package (4 dias / R$ 22). Esses serviços são salvos como catálogo ativo.

O campo "CEP de origem" é o endereço de onde seus pedidos saem. Use o CEP do seu depósito ou loja.

O que são transportadoras do catálogo

Transportadoras são as empresas que aparecem após a sincronização com a Frenet. Cada uma pode ter vários serviços (ex: PAC, SEDEX são serviços dos Correios).

Exemplo

Correios Total → PAC, SEDEX, SEDEX 10, SEDEX 12

Jadlog → Package, .Com, Cargo

Desativar uma transportadora aqui remove todos os seus serviços do checkout de uma vez. Para gerenciar serviço por serviço, use a seção "Serviços".

Como os serviços funcionam no checkout

Cada serviço de frete é uma opção que o cliente pode escolher no checkout. O sistema filtra automaticamente quais serviços são válidos para aquele pedido (considerando peso, dimensões e regras).

Exemplo

Produto pesa 500g, cliente em São Paulo. Frenet retorna PAC e SEDEX. Regra bloqueia SEDEX para essa categoria. Cliente vê apenas PAC.

Regras por categoria — menor prioridade

Regras de categoria se aplicam a todos os produtos de uma categoria. São a camada de menor prioridade: se existir uma regra de tipo ou produto para o mesmo serviço, ela vence.

Exemplo

Categoria "Eletrônicos" bloqueia SEDEX. Um produto eletrônico específico tem regra que permite SEDEX. O produto específico vence — SEDEX aparece para ele.

Regras de bloqueio sempre vencem regras de permissão dentro da mesma camada.

Regras por tipo — prioridade intermediária

Tipos logísticos são classificações que você atribui ao produto: "frágil", "perigoso", "refrigerado", "volumoso" etc. Regras de tipo se aplicam a todos os produtos com aquela classificação.

Exemplo

Tipo "Produto perigoso" bloqueia todos os serviços exceto Jadlog Package. Um produto classificado como perigoso, de qualquer categoria, só verá Jadlog Package no checkout.

Regras de tipo vencem categoria mas perdem para produto específico.

Regras por produto — máxima prioridade

Regras por produto específico têm precedência absoluta. Elas ignoram completamente as regras de tipo e categoria para aquele produto.

Quando usar

Produto "Mesa de vidro 2m" precisa de transportadora especial que não aparece para outros produtos da categoria "Móveis". Crie uma regra de produto para ele.

Use com moderação — ideal para exceções pontuais, não para regras gerais.

Entrega própria e retirada em loja

Estas opções são independentes da Frenet. Elas aparecem no checkout junto com os serviços normais, mas não passam pelo cálculo de frete da API.

Entrega própria

Você entrega com frota própria ou motoboy. Configure prazo e valor fixo ou grátis.

Retirada em loja

Cliente busca no endereço da loja. Normalmente frete R$ 0,00.

As regras de bloqueio por produto/tipo/categoria não afetam estas modalidades — elas seguem configuração própria.
